# PV Forecast Fusion MVP Implementation Plan

> **For Hermes:** Implement test-first. Every new behavior starts with a failing test.

**Goal:** Build a first Home Assistant custom integration that fuses multiple PV forecast sources into one explainable forecast.

**Architecture:** Use a small pure-Python fusion core plus a thin Home Assistant wrapper. The core handles source weighting, manual bias correction, confidence scaling, and weather-pattern classification from optional intraday curves. The HA layer reads configured sensors, feeds the core, and exposes fused sensors with diagnostics.

**Tech Stack:** Python 3.11+, Home Assistant custom integration structure, pytest, uv-managed local venv.

---

## MVP scope

1. Generic source slots instead of provider-specific hardcoding.
2. Per source:
   - name
   - today / tomorrow / remaining-today sensor
   - weight
   - bias factor
   - confidence
3. Fused output sensors:
   - today
   - tomorrow
   - remaining today
   - weather pattern
4. Diagnostic attributes with raw source values and contribution weights.
5. Pure unit tests for fusion math and curve/weather parsing.
6. HACS-ready repo scaffold.

## Post-MVP hooks

- Optional Solcast / Forecast.Solar presets in config flow
- Offset-in-minutes support for curve shifting
- Historical learning from actual production
- Per weather class bias profiles
